- Homebrew는 Mac에서 개발용 도구와 오픈소스 프로그램을 “의존성까지 포함해” 한 번에 설치·업데이트해 주는 패키지 관리자

## 오픈 소스 커뮤니티와 라이브러리

### 1) 오픈 소스 커뮤니티
- 누구나 코드를 공개하고
- 누구나 가져다 쓰고
- 누구나 개선에 참여할 수 있는 개발 생태계
    
### 2) 라이브러리
- 하나의 기능에 집중한 코드 묶음 즉 특정 기능을 담당하는 코드 묶음
- 예:
    - HTTP 요청 처리
        - GET/ POST 등
    - 이미지 변환
    - 비디오 인코딩

-  개발자는 모든 기능을 직접 만들지 않고 이미 만들어진 라이브러리를 조합해서 프로그램을 만듦
## 실제 상황
- 웹 브라우저를 만든다고 가정:
- HTTP 통신 라이브러리 필요
- 보안 라이브러리 필요
- 압축 라이브러리 필요

그런데 문제는: 
- A 라이브러리를 쓰려면 B, C가 필요하고
- B를 쓰려면 D가 필요하고
- 버전도 서로 맞아야 함
이걸 전부:
- 직접 찾고, 직접 다운로드하고, 직접 설치하고, 직접 업데이트해야 한다면  
→ 시간 낭비 + 오류 지옥

## homebrew가 해결하는 문제
- 자동설치
- 의존성 관리
- 업데이트 관리
- 설치된 모든 패키지를 최신 상태로 유지

- 개발 환경 세팅 시간 극단적으로 감소
- 팀 간 환경 차이 최소화
- 재설치, 이사, 새 mac에서도 동일한 환경 복구 가능
- 개발자가 도구 설치가 아니라 개발에 집중할 수 있는 환경을 만들어 줌
